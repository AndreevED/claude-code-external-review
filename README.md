# Claude Code External Review

Скиллы для Claude Code, которые отправляют артефакты (планы, код) на независимое ревью через внешнюю модель. Ревьюер исследует кодовую базу, возвращает замечания, Claude Code верифицирует и исправляет — цикл повторяется до одобрения.

## Скиллы

| Скилл | Назначение |
|-------|------------|
| `codex-plan-review` | Ревью планов реализации через Codex |

## Установка

Скопируйте нужные папки из `skills/` в директорию скиллов вашего проекта или глобально в `~/.claude/skills/`.

## codex-plan-review

Независимое ревью планов реализации через Codex.

### Использование

```bash
/codex-plan-review путь/к/плану.md [максимум N итераций]
```

Что происходит:
1. План и правила проекта отправляются Codex на ревью
2. Codex исследует кодовую базу и возвращает замечания (JSON)
3. Claude Code верифицирует каждое замечание по коду — подтверждает или отклоняет
4. Подтверждённые замечания исправляются в плане
5. Цикл повторяется до одобрения или лимита итераций (по умолчанию 5)

Результат: итоговый отчёт + лог в `.codex-plan-review/`.

### Ключевые особенности

**Верификация, а не слепое доверие.** Claude Code не принимает замечания Codex на веру — каждое проверяется по реальному коду. Замечание получает статус CONFIRMED (исправляется) или FALSE POSITIVE (отклоняется с обоснованием).

**Итеративный цикл.** Ревью — не одноразовый запрос. После исправлений план отправляется на повторное ревью, и так до одобрения или лимита итераций.

**Диалог между моделями.** Отклонённые замечания передаются обратно ревьюеру с развёрнутыми аргументами — почему отклонено, со ссылками на конкретный код. Ревьюер может принять аргумент или настоять на своём с новыми доказательствами.

**Накопление контекста между итерациями.** Каждая итерация передаёт ревьюеру списки уже принятых и отклонённых замечаний. Ревьюер не повторяет старое и сосредотачивается на новых проблемах.

**Детекция over-engineering.** Ревьюер отдельно оценивает избыточную сложность: лишние абстракции, преждевременная генерализация, ненужная индирекция, scope creep. Если план можно реализовать проще — это замечание.

**Порог уверенности.** Ревьюер сообщает только о проблемах с уверенностью >= 75. Стилистические придирки, предположения и «было бы неплохо» отсекаются на уровне системного промпта — в результате остаются только значимые замечания.

**Обнаружение тупика.** Если модели расходятся во мнениях и замечания начинают повторяться — цикл прерывается, пользователю показывается расхождение для ручного решения.

**Автоматическая сборка контекста.** Codex не видит `.claude/rules/` и `CLAUDE.md` — скилл собирает все правила (глобальные и проектные) и передаёт их явно в промпте, чтобы ревью учитывало стандарты проекта.

**Полный аудит.** Каждая итерация логируется: замечания, верификация, решения. Лог-файл — артефакт, к которому можно вернуться.

### Требования

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code)
- [Codex CLI](https://github.com/openai/codex) (команда `codex` в PATH)
